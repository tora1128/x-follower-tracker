# スマホだけで使う配布手順

スマホだけで完結させるには、このツールをインターネット上のサーバーに置いて、スマホからURLで開きます。

## 配布するもの

配布相手には、このGitHubリポジトリを渡します。

```text
https://github.com/tora1128/x-follower-tracker
```

または、READMEの `Deploy to Render` ボタンからデプロイしてもらいます。

配布元の `X_BEARER_TOKEN` は入れません。各利用者が自分のRenderにデプロイし、自分のX APIトークンを設定します。

利用者がデプロイした後は、利用者自身の公開URLをスマホで開きます。

```text
https://example.com/?key=WEB_APP_KEY
```

`WEB_APP_KEY` はサーバーに設定したアクセスキーです。URLを知っている人だけが操作できます。

## サーバーに設定する環境変数

```env
X_BEARER_TOKEN=your_bearer_token_here
X_USER_ID=123456789
WEB_APP_KEY=long_random_key
XFT_STORAGE_DIR=/data
```

`X_USER_ID` がわからない場合は、`X_USERNAME` を使えます。

```env
X_BEARER_TOKEN=your_bearer_token_here
X_USERNAME=example_user
WEB_APP_KEY=long_random_key
XFT_STORAGE_DIR=/data
```

## 起動コマンド

```bash
python3 web_app.py
```

クラウド環境では `PORT` が自動設定される場合があります。このツールは `PORT` 環境変数を自動で読みます。

## Renderで公開する場合

このフォルダには `render.yaml` を入れてあります。Renderでは Blueprint として作成すると、Webサービス、環境変数入力、永続ディスクをまとめて設定できます。
カード登録なしで試せるよう、標準設定は無料プラン・永続ディスクなしにしています。

設定する値:

- `X_BEARER_TOKEN`: X APIのBearer Token
- `X_USER_ID`: 対象アカウントのユーザーID
- `X_USERNAME`: `X_USER_ID` を使わない場合だけ設定
- `WEB_APP_KEY`: スマホURLに付ける長いアクセスキー

公開後、RenderのURLに `?key=WEB_APP_KEY` を付けて配布します。

```text
https://your-service.onrender.com/?key=WEB_APP_KEY
```

## 永続保存について

`data/` と `reports/` は `XFT_STORAGE_DIR` の下に作られます。クラウドで使う場合は、永続ディスクを `XFT_STORAGE_DIR` に指定してください。

無料プランの標準設定では `XFT_STORAGE_DIR=/tmp/x-follower-tracker` を使います。この場合、サーバー再起動や再デプロイで保存データが消える場合があります。

データを消したくない場合は、有料プランで永続ディスクを追加し、`XFT_STORAGE_DIR` をそのディスクのパスに変更してください。

## Dockerで動かす場合

```bash
docker build -t x-follower-tracker .
docker run -p 8000:8000 \
  -e X_BEARER_TOKEN=your_bearer_token_here \
  -e X_USER_ID=123456789 \
  -e WEB_APP_KEY=long_random_key \
  -v xft-data:/data \
  x-follower-tracker
```

公開後は次のURLをスマホで開きます。

```text
http://サーバーのURL:8000/?key=long_random_key
```
